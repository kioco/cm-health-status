cm-health-status
================
This project provides a simple utility to poll the Health Status Checks for the various entities within Cloudera Manager.

A brief note, to better understand the Entity Model of CM:

- CM can manage a list of Clusters
    - Each Cluster has a list of hosts, and services
        - Each Host has a list of health checks which can be performed upon it, along with a summary which indicates the aggregate health
        - Each Service has a list of health checks which can be performed upon it, along with a summary which indicates the aggregate health
            - Each Service has a list of roles
              - Each Role has a list of health checks which can be performed upon it, along with a summary which indicates the aggregate health

This utility creates a listing of all these checks and their corresponding status.

# Prereqs
```sh
# Install the Python bindings for the CM API
$ sudo pip install cm-api
```

# Sample Usage
```sh
# "user:pass" inside a pass_file
$ echo "admin:admin" > pass.file
# ./health.py -H <host> -p <port> -P <password_file>
$ ./health.py -H nightly52-1.ent.cloudera.com -p 7180 -P pass.file
```
